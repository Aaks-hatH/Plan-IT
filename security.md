## Reporting a Security Vulnerability

If you discover a security vulnerability in PlanIt, **please do not open a public
GitHub issue**. Public disclosure before a fix is in place puts all PlanIt users
at risk.

**Report vulnerabilities privately to:**

 planit.userhelp@gmail.com  
**Subject line:** `[SECURITY] Vulnerability Report — <brief description>`

Please include:

- A clear description of the vulnerability
- Steps to reproduce it
- The potential impact (what an attacker could do)
- Any proof-of-concept code or screenshots (if applicable)

We will acknowledge your report within **48 hours** and aim to issue a fix or
mitigation within **14 days** for critical issues.

---

## Scope

The following are **in scope** for security reports:

- The hosted service at [planitapp.onrender.com](https://planitapp.onrender.com)
- Authentication and authorization bypasses
- Data exposure or exfiltration vulnerabilities
- XSS, CSRF, SQLi, or injection vulnerabilities in the hosted service

The following are **out of scope:**

- Vulnerabilities in self-hosted deployments (self-hosting is not permitted)
- Social engineering attacks
- Denial of service via volumetric traffic
- Reports generated by automated scanners without manual validation

## Security Testing

PlanIt performs ongoing security testing and review processes intended to identify,
validate, and remediate vulnerabilities across the platform and supporting
infrastructure. Security testing activities may include, but are not limited to:

- Manual application security assessments
- Authentication and authorization testing
- Static and dynamic analysis
- Dependency and supply-chain review
- Infrastructure and configuration auditing
- Session management validation
- Input validation and injection testing
- Logging, monitoring, and incident response review

Security controls and remediation priorities are continuously evaluated based on
risk, exploitability, and potential user impact.

Testing methodologies may align with industry-standard practices including the
OWASP Web Security Testing Guide (WSTG), OWASP ASVS, and applicable secure
development lifecycle principles.

While reasonable efforts are made to secure the platform, no system can be
guaranteed to be completely secure. Users are responsible for maintaining the
security of their own devices, credentials, and access environments.


### Authorized Security Research

PlanIt may permit limited, authorized security testing by qualified security
researchers, professional penetration testers, or red team operators on a
case-by-case basis.

Organizations or individuals with demonstrable professional experience who wish
to conduct coordinated security assessments must obtain explicit written
authorization before performing any testing against PlanIt systems, services,
or infrastructure.

Requests for authorized testing should include:

- Full name and organization
- Scope of proposed testing
- Testing methodology and tools
- Intended testing dates and duration
- Relevant certifications, prior work, or professional references

Unauthorized testing remains strictly prohibited.

To request authorization, contact:

planit.userhelp@gmail.com  
**Subject line:** `[SECURITY] Authorized Testing Request`
